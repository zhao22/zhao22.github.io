继承初探

继承属于类与类之间关系的一种。继承自其它类的类叫做子类，被其他类继承的类叫做父类。
比如猫和动物，猫属于动物的一种，那么猫就可以继承自动物，此时动物就是父类，猫就是子类。

继承

我们使用继承的目的主要有两个，一个是利用继承使子类复用父类的属性和方法，另外是用于向上造型。

继承的关系

  1. java与C++不同，java中的继承是单继承:
    一个类只能继承自一个父类。
    比如猫类不能同时继承自动物类和猫科动物类。

  2. 一个类可以被多个类继承:
    比如动物类可以同时被猫类，狗类…同时继承，子类都可以调用动物类的方法。

  3. 继承是链式的，即一个类可以调用它父类的父类…的方法:
    在介绍单继承的时候，我们举了个例子:
    猫类不能同时继承自动物类和猫科动物类
    但这个例子我们可以稍加改装，将猫类继承自猫科动物类，再让猫科动物类继承自动物类，这时我们就可以在猫类同时调用猫科动物类和动物类的方法了。

重写(overwrite)

在父类的方法无法满足子类的要求时，我们可以对父类方法进行重写。
重写的方法必须满足以下条件:

  1. 子类方法的修饰符范围必须大于等于父类方法的修饰符范围。
    范围规则是:public > protected > ‘’ > private, 如果父类方法的范围是protected，子类就只能是protected或者public。
  2. 子类方法的返回值类型必须与父类方法相同。
    理由和重载时一致，编译期无法通过返回值类型来判断具体选择哪个方法。
  3. 子类方法的方法名必须与父类方法相同。
  4. 子类方法的参数类型必须与父类相同。
    这里如果子类的参数类型与父类不同，那么这个方法和父类的同名方法构成重载，不会报错。

只有满足以上条件，子类方法才能重写父类方法。
