什么是内存

不了解计算机的人可能会把内存和硬盘空间混为一谈。

内存(RAM)，通常是作为操作系统或其他正在运行程序的临时存储介质，也称作系统内存。就好比计算机中的内存条，如果内存条容量越大，计算机就有更多的内存来存储同时运行的任务，这样系统响应的速度也就越快。

硬盘空间(ROM)，主要包含自身系统占据的空间和用户可用的空间两部分，用来存储和保存数据。即使是断电，ROM也能够保留数据。

所以java程序在运行过程中，编译后的类和方法，以及我们的变量，对象等等都是放入内存的。如果停止程序或者断电，这些记录就会消失。而如果要长久保存，我们需要将数据导出到文件或者其他存储介质。

这就是java的值传递。

简单地说，在将一个参数传给方法时，只是将这个参数的值赋予给了形式参数。
刚刚这段程序，在将arg传给addOne的时候，只是将arg的值1传给了方法。在方法内形参arg的值增加了1，但方法外实际arg的值没有变。
而在将student传递给方法的时候，是将student的地址传递给了形参student。此时形参和实际参数都是指向的堆中的同一个对象。

java内存回收机制

java的虚拟机jvm是提供了垃圾回收机制GC(Gabagge Collection).GC是一个独立于程序运行的线程，会将没有被引用指向的对象清理掉，这样java就不用手动关闭指针，但它有自己的回收算法，不能保证什么时候回收哪些变量。如果要加快回收的到来，可以使用System.gc()，但我们还是应该合理控制内存空间。
有的类，如Scanner，它的内部实现导致有的引用始终指向对象，导致对象无法被回收。因此我们应该在使用完Scanner后，调用它的close()方法，以免内存泄露。
