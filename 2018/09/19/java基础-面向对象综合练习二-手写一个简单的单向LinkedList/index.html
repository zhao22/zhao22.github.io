什么是链表

链表是用于储存一组数据的数据结构。但它的元素在内存中的位置是任意的，单向链表的第一个元素除了储存本身应该储存值以外，还会存储第二个元素的地址。第二个元素除了存储本身的值以外，还会存储第三个元素的地址…直到链表的最后一个元素，它的下一个元素为空,这些地址就像一条条锁链，将物理位置不连续的元素连在了一起，组成了一个数据组合。

链表的常用操作的思想

遍历

从链表的第一个元素A开始，如果这个元素有下一个元素B，那么就继续找元素B的下一个元素C…直到找到一个元素X再没有下一个元素，那么当前这个元素X就是链表的最后一个元素。

末尾插入

如果链表中还没有元素，那么链表的第一个元素就是当前插入的元素。
否则遍历链表，将末尾元素的下一个元素设置为当前插入的元素，就可以完成末尾插入。

开头插入

将插入的元素作为链表的第一个元素，将原本链表的第一个元素作为插入元素的下一个元素。

查找位置为index的元素

从链表第一个元素A开始,找下一个元素B，再找下一个元素C…直到查找次数等于index时，返回该节点。

在index的位置插入

找到位于index - 1的节点A，它的下一个元素就是原本位于index的元素B.我们将节点A的下一个元素设为插入节点，再将插入节点的下一个元素设为B，就完成了插入。

删除位于开头的元素

将链表的第一个元素设为当前链表第一个元素的下一个元素。

删除位于末尾的元素

找到末尾前的一个元素A，它原本下一个元素B是需要删除的元素.将A的下一个元素设为null。就将B抛出了链表。

删除index位置的元素

找到位于index - 1的节点A，它的下一个元素就是应该删除的元素B.B的下一个元素是C.将A的下一个元素设为C.就将B抛出了链表。

链表和数组的优缺点对比

数据结构 数组 链表
内存大小 只用储存当前元素，较小 不仅需要储存当前元素，还要储存下一个元素的位置，较大
遍历 物理结构顺序遍历，较快 地址寻址遍历，较慢
插入 需要为数组扩容，且需要遍历将后续数据后移，较慢 只需要变换一下指针，非常快
删除 需要遍历将后续数据前移，且需要为数组缩容，较慢 只需要变换一下指针，非常快

由这个对比我们可以看出，如果一个列表元素经常变化，那么优先选择链表。而如果一个列表变化较小，但经常查找数据，那么优先选择数组。

实现链表

我们通过分析发现，链表中的每个元素都需要储存一个值和下一个元素，这在当前的数据结构里是没有的。我们想到可以通过构造一个类来作为链表中的元素类。每一个对象对应链表中的一个元素。而这个类只提供链表使用，不对外部开放，所以我们可以通过内部类来实现。
